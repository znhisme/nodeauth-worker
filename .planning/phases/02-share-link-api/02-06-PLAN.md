<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Source -> generated backend bundles | Security fixes in `src/**` must reach Worker, Docker, and Netlify runtime artifacts. |
| Browser origin -> generated `/api/*` CORS middleware | Deployed bundles must not let arbitrary origins read credentialed authenticated API responses. |
| Public recipient -> generated `resolveShareAccess()` | Deployed bundles must not decrypt vault secret material or generate OTP output before access-code verification. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06-01 | Tampering | `backend/dist/**` generated artifacts | mitigate | Rebuild all generated bundles from source with project scripts and verify source maps; do not hand-edit generated files. |
| T-02-06-02 | Information Disclosure | Worker/Docker/Netlify API CORS middleware | mitigate | Grep and Node assertions require `resolveApiCorsOrigin` and reject `origin: (origin) => origin` in every runtime bundle. |
| T-02-06-03 | Information Disclosure | Worker/Docker/Netlify public share access | mitigate | Node assertions require `verifyShareSecret` before `decryptField` and OTP `generate(` inside generated `resolveShareAccess()` for every target. |
| T-02-06-04 | Denial of Service | Build/source-map verification pipeline | accept | Build failure blocks this plan and prevents stale bundle publication; no additional runtime mitigation is needed in a distribution refresh plan. |
</threat_model>
