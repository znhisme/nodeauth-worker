<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| owner client -> `/api/share/*` | Authenticated owner JSON and path params enter share management routes. |
| public recipient -> `/api/share/public/:token/access` | Unauthenticated token path and access-code body enter public route. |
| route -> service | HTTP parsing and auth context become typed service inputs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Elevation of Privilege | Owner route handlers | mitigate | Apply `authMiddleware` directly to `POST /`, `GET /`, `GET /:id`, and `DELETE /:id`; derive ownerId as `user.email || user.id`. |
| T-02-02-02 | Spoofing | Public access-code submission | mitigate | Require access code through JSON body `accessCode`; never read `accessCode` from URL query strings. |
| T-02-02-03 | Denial of Service | Public token/code brute force | mitigate | Attach `shareRateLimit()` to `POST /public/:token/access` before `resolveShareAccess()`. |
| T-02-02-04 | Information Disclosure | Public inaccessible responses | mitigate | Collapse missing, invalid, expired, revoked, deleted, locked, rate-limited, middleware-error, and wrong-code states to `{ success: false, message: 'share_inaccessible', data: null }` status 404. |
| T-02-02-05 | Information Disclosure | Public response headers | mitigate | Apply `Cache-Control: no-store`, `Pragma: no-cache`, and `Referrer-Policy: no-referrer` to every public response from both `shareRoutes.ts` and blocking paths inside `shareRateLimitMiddleware.ts`. |
| T-02-02-06 | Information Disclosure | Owner route responses | mitigate | Return only owner DTOs from Plan 01; route tests serialize responses and reject raw tokens except create, hashes, session fields, and credentials. |
</threat_model>
