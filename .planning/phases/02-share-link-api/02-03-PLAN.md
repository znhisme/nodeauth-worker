<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request -> root app | Public token-bearing paths enter global middleware and logging. |
| root app -> share route module | Requests under `/api/share` are dispatched after global initialization, CORS, secure headers, and health gate. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Information Disclosure | Hono global logger | mitigate | Redact `/api/share/public/:token/access` to `/api/share/public/[share-token]/access` before `logger.info`. |
| T-02-03-02 | Tampering | Route mounting order | mitigate | Mount `app.route('/api/share', shareRoutes)` before `app.all('/api/*')` so valid share APIs are not swallowed by the 404 handler. |
| T-02-03-03 | Denial of Service | Health gate bypass | mitigate | Do not exempt `/api/share` from `runHealthCheck`; unsafe deployments still block sensitive share APIs. |
| T-02-03-04 | Information Disclosure | Unknown API route handling | accept | Existing `app.all('/api/*')` generic 404 remains after mounted routes; no share-specific secret material is included. |
</threat_model>
