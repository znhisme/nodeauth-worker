<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| unauthenticated public recipient -> share route | Public token/code requests must not reveal state distinctions. |
| unauthenticated public recipient -> rate limiter | Lockout/fail-closed responses happen before route handler execution. |
| request path -> centralized logs | Raw bearer-like public share URL tokens can appear in request paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T3-15 | Information Disclosure | token enumeration | mitigate | Missing-token/hash tests assert identical generic inaccessible decisions and no share data. |
| T3-16 | Spoofing | wrong-code handling | mitigate | Service tests assert wrong-code path rejects before secret processing and remains generic. |
| T3-17 | Information Disclosure | revoked/expired/deleted-item privacy | mitigate | Route and service tests assert same public envelope for revoked, expired, deleted, and missing states. |
| T3-18 | Denial of Service | locked/rate-limited state | mitigate | Middleware tests assert locked/fail-closed paths return same generic body and public headers. |
| T3-19 | Information Disclosure | response DTO leakage | mitigate | Exact key allowlists for owner metadata and public `SharedItemView`. |
| T3-20 | Information Disclosure | log redaction | mitigate | App tests assert raw public share path tokens are masked before logger use. |
</threat_model>
