<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source -> generated backend bundles | Cloudflare, Docker, and Netlify deploy generated artifacts, not just `src/**`. |
| generated bundles -> public share recipients | Rebuilds must preserve generic errors and no-store/no-referrer protections for sensitive shared credentials. |
| generated source maps -> provenance | Source-map drift can hide regenerated-output/source mismatch. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T3-44 | Tampering | `backend/dist/**` generated artifacts | mitigate | Rebuild only through `build:worker`, `build:docker`, and `build:netlify`; never hand-edit bundles. |
| T3-45 | Denial of Service | MySQL runtime schema in generated bundles | mitigate | Grep all bundles for `VARCHAR(64)` and fail on legacy `VARCHAR(36)` share/audit ID definitions. |
| T3-46 | Information Disclosure | public share response contract | mitigate | Generated assertions require `share_inaccessible`, `Cache-Control`, and `Referrer-Policy` after rebuild. |
| T3-47 | Tampering | source-map provenance | mitigate | Run source-map verification and refresh provenance only through `restore_backend_source_from_sourcemaps.js` when required. |
| T3-48 | Denial of Service | existing NodeAuth behavior | mitigate | Run full `npm --prefix backend test` before and after bundle regeneration. |
</threat_model>
