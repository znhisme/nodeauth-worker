<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source-map -> editable source | Generated source maps are trusted only as checked-in provenance for restoring editable TypeScript files. |
| editable source -> generated bundles | Build scripts regenerate `backend/dist/**`; generated output is not hand-edited. |
| package registry -> backend install | `backend/package-lock.json` pins backend dependencies before security-sensitive code is added. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `backend/dist/**` | mitigate | Restore `src/**`, create build scripts, and verify source with `node scripts/restore_backend_source_from_sourcemaps.js --verify`; generated bundles are not primary implementation files. |
| T-01-02 | Tampering | backend dependency install | mitigate | Generate `backend/package-lock.json` and use `npm ci --prefix backend` for reproducible installs. |
| T-01-03 | Information Disclosure | frontend source availability | accept | Editable frontend source is absent; record API-only UX-04 scope in `.planning/source-provenance.md` and do not edit `frontend/dist/**`. |
| T-01-04 | Tampering | generated-bundle drift | mitigate | Build Worker, Docker, and Netlify outputs from restored source before downstream share changes. |
| T-01-05 | Tampering | cross-runtime source drift | mitigate | Use one restored `src/**` tree and three explicit build scripts rather than per-runtime hand patches. |
</threat_model>
