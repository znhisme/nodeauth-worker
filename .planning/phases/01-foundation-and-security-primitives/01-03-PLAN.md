<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| service -> database | Share state, audit events, and rate-limit counters cross into durable storage. |
| owner identity -> vault item | Owner id must be checked against vault item accessibility before share creation. |
| recipient public input -> service | Raw token/access code are untrusted and must become derived values before lookup/verification. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-12 | Information Disclosure | raw token leakage | mitigate | `ShareService.createShare` stores only `tokenHash` and returns raw token only in `CreateShareResult`; tests assert repository input lacks raw token fields. |
| T-01-13 | Spoofing | access-code guessing | mitigate | `shareRateLimit` fails closed on missing DB, repository error, denied decision, or lock state with generic `share_inaccessible`. |
| T-01-14 | Denial of Service | fail-open rate limiting | mitigate | Do not reuse `rateLimit()`; `shareRateLimitMiddleware.ts` throws before `next()` when storage cannot enforce limits. |
| T-01-15 | Elevation of Privilege | deleted/revoked/expired share bypass | mitigate | `ShareService.resolveShareAccess` checks token hash, access code hash, `expiresAt`, `revokedAt`, and active vault item before returning accessible. |
| T-01-16 | Information Disclosure | raw TOTP seed disclosure | mitigate | `SharedItemView` allowlist from Plan 02 remains the only recipient DTO contract; Phase 1 service does not expose raw seed or otpauth URI. |
| T-01-17 | Information Disclosure | unsafe logging/audit data | mitigate | `ShareRepository.insertAuditEvent` accepts only safe audit event fields and tests reject raw token/code/password/seed fields. |
| T-01-18 | Elevation of Privilege | cross-owner access | mitigate | `VaultRepository.findActiveByIdForOwner` requires undeleted item and `createdBy` null or matching owner id. |
| T-01-19 | Tampering | cross-runtime schema drift | mitigate | Add identical share table/index names to SQLite, MySQL, PostgreSQL Drizzle schemas, `backend/schema.sql`, and runtime migrator version 13. |
</threat_model>
