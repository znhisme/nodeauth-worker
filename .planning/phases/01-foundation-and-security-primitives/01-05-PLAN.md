<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public recipient -> share limiter | Unauthenticated requests provide a bearer URL token and eventually access-code attempts. Raw token material must not be durably persisted outside one-way derivations. |
| share service -> public route serialization | Public route code may serialize `ShareAccessDecision`; the decision object must be safe by construction. |
| owner request -> revoke repository/audit | Authenticated owner-controlled share IDs cross into repository updates and audit insertion. No-op revokes must not create misleading audit events. |
| source -> generated backend bundles | Source fixes must be reflected in Cloudflare Worker, Docker, and Netlify outputs before deployment. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05-01 | I | `shareRateLimitMiddleware.ts` | mitigate | Hash the raw URL token with `hashShareSecret(getShareSecretPepper(env), 'share-token', rawToken)` before building `key` or `shareId`; tests assert the raw token is absent. |
| T-01-05-02 | I | `ShareService.resolveShareAccess()` / `ShareAccessDecision` | mitigate | Remove `ShareLinkRecord` from public decisions; tests serialize allowed and denied decisions and assert forbidden fields are absent. |
| T-01-05-03 | R | `ShareRepository.revokeForOwner()` / revocation audit | mitigate | Return false for missing, wrong-owner, and already-revoked shares; `ShareService.revokeShare()` keeps throwing and does not insert audit on false. |
| T-01-05-04 | I | public response headers | mitigate | Include `getSharePublicHeaders()` on denied and allowed `resolveShareAccess()` outcomes so route serialization inherits no-store/no-referrer protections. |
| T-01-05-05 | T | `backend/dist/**` generated outputs | mitigate | Rebuild Worker/Docker/Netlify bundles from source and verify with source-map and schema alignment commands. |
</threat_model>
