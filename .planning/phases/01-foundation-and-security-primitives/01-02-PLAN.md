<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| owner browser -> create-share contract | Raw token/access code can be returned only once after creation. |
| recipient browser -> public share access | Access code must cross in a non-URL request body and receive no-store/no-referrer protections. |
| application -> database | Database stores only HMAC-derived token and access-code values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Information Disclosure | raw share token | mitigate | `generateShareToken()` produces 32 random bytes; contract requires raw token returned once and stored only as HMAC-SHA-256. |
| T-01-07 | Spoofing | access-code guessing | mitigate | `generateAccessCode()` produces 16 random bytes; contract requires body submission and share-specific fail-closed limiter in Plan 03. |
| T-01-08 | Information Disclosure | raw TOTP seed disclosure | mitigate | `SharedItemView` allowlist excludes seed and otpauth URI; contract allows only current OTP code/countdown. |
| T-01-09 | Information Disclosure | cache/referrer leakage | mitigate | `getSharePublicHeaders()` returns `Cache-Control: no-store`, `Pragma: no-cache`, and `Referrer-Policy: no-referrer`. |
| T-01-10 | Information Disclosure | unsafe logging/audit data | mitigate | Contract forbids raw tokens, access codes, full URLs, passwords, TOTP seeds, and OTP codes in audit/log records. |
| T-01-11 | Spoofing | canonical public origin confusion | mitigate | `normalizePublicOrigin()` accepts `https://` and localhost HTTP origins only; unsafe origins throw `invalid_public_origin`. |
</threat_model>
