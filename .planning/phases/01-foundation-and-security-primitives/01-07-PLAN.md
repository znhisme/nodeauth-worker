<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public recipient -> `ShareService.resolveShareAccess()` | Unauthenticated input supplies a bearer URL token and access code; service decisions and audit writes must not expose raw secrets or internal share rows. |
| public recipient -> `shareRateLimit()` | Unauthenticated repeated attempts cross into durable rate-limit and audit storage; limiter keys and audit metadata must use derived/sanitized values only. |
| service/middleware -> `share_audit_events` | Security event metadata is durable internal evidence; it must remain useful without storing raw tokens, access codes, passwords, TOTP seeds, otpauth URIs, or full public URLs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07-01 | R | `ShareService.resolveShareAccess()` successful access | mitigate | Insert `access_granted` with `actorType: 'recipient'`, `shareId`, `ownerId`, `eventAt`, and minimal metadata after successful access-code verification; tests assert forbidden values are absent. |
| T-01-07-02 | R | `ShareService.resolveShareAccess()` expiration transition | mitigate | Insert `expired` with `actorType: 'system'` when an active share row has `expiresAt <= now`; tests assert the public decision remains generic and metadata is safe. |
| T-01-07-03 | R | `shareRateLimit()` denied threshold | mitigate | On `decision.allowed === false`, resolve the share by derived `tokenHash` and insert `access_denied_threshold`; skip audit for unknown token hashes rather than fabricating required owner/share IDs. |
| T-01-07-04 | I | audit metadata, logs, limiter keys, test snapshots | mitigate | Explicitly prohibit raw tokens, access codes, passwords, TOTP seeds, otpauth URIs, session cookies, owner email, and full public URLs; tests serialize audit inputs and limiter inputs and assert forbidden substrings are absent. |
| T-01-07-05 | D | share limiter storage/audit failure | mitigate | Preserve fail-closed middleware behavior: storage, lookup, hashing, or audit failures still return generic `share_inaccessible` and do not call `next()`. |
</threat_model>
